# Books AI — Project Context for Claude Code

## What This Project Does

Books AI is a full-stack AI-powered book discovery and recommendation platform. Users interact via a natural language chat interface — "recommend something like Dune", "add this to my shelf", "write a review for Project Hail Mary". An AI agent orchestrator handles all intent routing, RAG retrieval, and MCP tool dispatch.

## Architecture Decisions

- **Agent orchestration**: LangGraph `StateGraph` — NOT LangChain chains. Always use typed `AgentState` TypedDict for state.
- **LLM**: `claude-sonnet-4-20250514` for all agent calls. Use streaming everywhere user-facing.
- **MCP protocol**: All agent→tool boundaries go through MCP. Never call microservices directly from the agent layer.
- **RAG**: pgvector (not Qdrant) for under 5M embeddings. Switch to Qdrant via `rag/utils/db_factory.py` when scale demands.
- **DB**: PostgreSQL 16 for all relational + vector data. Alembic for migrations. Never raw SQL in service code — use SQLAlchemy ORM.
- **Async**: All Python code is `async`/`await` throughout. No synchronous IO in hot paths.
- **Transport**: gRPC between internal services for latency-sensitive paths (recommendations), REST for everything else.

## Coding Standards

### Python (agents, mcp-server, rag, services)
- Python 3.12+
- Type hints on every function — no `Any` unless truly unavoidable
- Pydantic v2 models for all request/response schemas
- `ruff` for linting, `black` for formatting (line length 100)
- Docstrings on every public function and class (Google style)
- No print statements — use structured logging (`structlog`)
- All tests with `pytest` + `pytest-asyncio`

### TypeScript (ui)
- Strict mode enabled (`"strict": true` in tsconfig)
- No `any` types
- `eslint` + `prettier` enforced in CI
- React Server Components by default; `"use client"` only when needed
- `zod` for all runtime validation of API responses

### General
- Every module has a `__init__.py` (Python) or `index.ts` (TS) that re-exports public API
- No circular imports — enforced by `import-linter` (Python) and `eslint-plugin-import` (TS)
- Every new feature needs unit tests before merge

## Running Locally

```bash
# Start all services via Docker Compose
docker compose -f infra/docker/docker-compose.dev.yml up -d

# Run the UI
cd ui && npm install && npm run dev

# Run the agent (hot-reload)
cd agents && uvicorn main:app --reload --port 8000

# Run the MCP server
cd mcp-server && uvicorn main:app --reload --port 8001
```

## Key Domain Concepts

- **Book**: Core entity. Has title, author, ISBN, genres (array), summary, cover_url
- **Shelf**: A named reading list owned by a user (e.g. "Want to read", "Favourites")
- **Review**: A 1–5 star rating + optional text body. One per user per book.
- **Recommendation**: A ranked list of books generated by the AI layer, not stored permanently
- **Embedding**: A 1536-dim vector representation of a book's summary chunk, stored in `book_embeddings`

## What Claude Code Should NOT Do

- Never modify `db/migrations/` directly — always use `alembic revision --autogenerate`
- Never add new environment variables without updating `.env.example`
- Never bypass the MCP layer to call microservices directly from agents
- Never use `time.sleep()` — use `asyncio.sleep()`
- Never commit secrets, API keys, or connection strings
- Never use `SELECT *` in production queries — always name columns
